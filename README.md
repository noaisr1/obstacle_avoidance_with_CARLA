# Obstacle Avoidance with Semantic Segmentation in CARLA

## Overview
This project implements a simple obstacle avoidance and safety supervision system in the CARLA simulator.

An ego vehicle drives using CARLA's built-in autopilot, while a Safety Supervisor monitors a semantic segmentation camera and applies emergency braking when obstacles are detected in front of the vehicle.

The project is designed as a baseline for a Master's final project and focuses on perception, decision-making, and evaluation rather than navigation.

---

## Project Structure

project_obstacle_avoidance/
- main.py                Main simulation loop (autopilot + supervisor)
- config.py              Global configuration parameters
- sim_setup.py           CARLA connection, vehicle and sensor setup
- perception.py          Semantic segmentation based obstacle detection
- supervisor.py          Safety decision logic (emergency braking)
- metrics.py             CSV logging of driving and safety metrics
- analysis.py            Offline analysis and graph generation
- run_metrics.csv        Output data from a simulation run
- analysis_results/      Saved graphs (generated by analysis.py)

---

## Requirements

- Windows
- CARLA Simulator (tested with CARLA 0.9.15)
- Python 3.7
- Python virtual environment (venv)
- Python packages:
  - numpy
  - matplotlib

CARLA must be installed separately and running before executing the code.

---

## Environment Setup (One-Time)

1. Install CARLA and note the path to:
   C:\CARLA\WindowsNoEditor

2. Create and activate a Python 3.7 virtual environment (example):
   C:\venvs\carla37

3. Install required Python packages:
   pip install numpy matplotlib

4. Verify that `CARLA_ROOT` in `config.py` points to:
   C:\CARLA\WindowsNoEditor

---

## Running the Simulation

1. Start CARLA:
   C:\CARLA\WindowsNoEditor\CarlaUE4.exe

2. Open PowerShell and activate the virtual environment:
   C:\venvs\carla37\Scripts\Activate.ps1

3. Navigate to the project directory:
   cd C:\CARLA\projects\project_obstacle_avoidance

4. Run the simulation:
   python main.py

5. When the run finishes, a file named `run_metrics.csv` will be created.

To stop the simulation manually:
- Press Ctrl+C in the terminal.

---

## Running Experiments

The experiment mode is controlled via `config.py`:

RUN_MODE = "BASELINE"  
- Ego vehicle drives using autopilot only.

RUN_MODE = "SUPERVISOR"  
- Ego vehicle drives with autopilot and safety supervisor override.

Suggested workflow:
1. Run once with BASELINE and save the CSV as `run_metrics_baseline.csv`.
2. Run again with SUPERVISOR and save the CSV as `run_metrics_supervisor.csv`.
3. Compare the results.

---

## Data Analysis and Graphs

To generate graphs from the recorded data:

1. Make sure `run_metrics.csv` exists.
2. Run:
   python analysis.py

3. The following graphs will be saved in `analysis_results/`:
   - obstacle_ratio_over_time.png
   - speed_with_safety_overrides.png

---

## Output Metrics

The CSV file contains:
- time_sec: elapsed time since start
- frame: CARLA frame id
- speed_mps: vehicle speed
- obstacle_ratio: percentage of obstacle pixels in ROI
- override: whether safety supervisor intervened
- brake: braking intensity
- collision: collision event indicator

---

## Notes

- The project intentionally avoids CARLA navigation agents to reduce dependencies.
- Built-in CARLA autopilot is used for stable baseline behavior.
- The system is conservative by design and may stop permanently after detecting obstacles.
- This behavior is acceptable and discussed as a design choice in the analysis.
